# 📁 STEP 1: COMPLETE PROJECT STRUCTURE

The Ecom.Reporting solution follows Clean Architecture principles, ensuring a separation of concerns that makes the system maintainable, testable, and scalable.

Ecom.Reporting
├── Ecom.Reporting.API            [Presentation Layer]
├── Ecom.Reporting.Application    [Business Logic Layer]
├── Ecom.Reporting.Domain         [Core Domain Layer]
├── Ecom.Reporting.Infrastructure [Implementation/Data Layer]
├── Ecom.Reporting.Tests          [Quality Assurance Layer]
└── Dockerfile                     [Containerization]

### 🔹 Layer Responsibilities:

1.  **Ecom.Reporting.Domain**: 
    - **Why it exists**: To hold the "Heart of the Application" — business entities and rules.
    - **Responsibility**: Pure logic, zero dependencies on DB, frameworks, or APIs.
    - **Dependency Direction**: Inward. No other project depends on anything but the Domain.

2.  **Ecom.Reporting.Application**:
    - **Why it exists**: To orchestrate business use cases.
    - **Responsibility**: Defines "What" the system does. Contains Interfaces (abstractions), DTOs, and Services.
    - **Dependency Direction**: Depends ONLY on the Domain.

3.  **Ecom.Reporting.Infrastructure**:
    - **Why it exists**: To provide technical implementations (Database, Messaging).
    - **Responsibility**: Implements the interfaces defined in Application. Handles EF Core, RabbitMQ, etc.
    - **Dependency Direction**: Depends on Application and Domain.

4.  **Ecom.Reporting.API**:
    - **Why it exists**: To expose the application via HTTP.
    - **Responsibility**: Handles routing, controllers, middleware, and entry points.
    - **Dependency Direction**: Depends on Application and Infrastructure (for Wiring/DI).

5.  **Ecom.Reporting.Tests**:
    - **Why it exists**: To ensure correctness.
    - **Responsibility**: Unit and Integration tests.

## 🏗️ DOMAIN LAYER (CORE)

### 🔹 File Name: AuditLog.cs
**Location**: `Ecom.Reporting.Domain/Entities/AuditLog.cs`
**Purpose**: The central entity representing a recorded event in the system.

**FULL CODE**:
```csharp
namespace Ecom.Reporting.Domain.Entities;

public class AuditLog
{
    public int Id { get; set; }
    public string? EntityName { get; set; }
    public int? EntityId { get; set; }
    public string? Action { get; set; }
    public string? EventType { get; set; }
    public string? SourceService { get; set; }
    public string? CorrelationId { get; set; }
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
}
```

**LINE-BY-LINE EXPLANATION**:
- `public class AuditLog`: Defines a blueprint for an audit record. If removed, the system lacks a way to store "What happened".
- `public int Id { get; set; }`: The unique fingerprint (Primary Key). Used by DB to fetch specific logs.
- `public string? EntityName`: Stores the type of object changed (e.g., "Product"). Essential for filtering logs by category.
- `public int? EntityId`: Stores the specific ID of the object. Vital for tracing the history of a single item.
- `public string? Action`: The human-readable action (e.g., "Approved"). If removed, you wouldn't know "What" was done.
- `public string? EventType`: The technical event name (e.g., "product.approved"). Bridges logic to messaging.
- `public string? SourceService`: Tracks which microservice sent the data. Crucial for debugging "Who triggered this".
- `public string? CorrelationId`: A unique ID shared across services. If removed, you cannot trace a single transaction across the whole ecosystem.
- `public DateTime CreatedAt = DateTime.UtcNow`: Records "When". Defaults to UTC to avoid time-zone chaos.

**REAL-WORLD ANALOGY**:
Think of a **Black Box Flight Recorder**. It doesn't fly the plane; it just records every single switch flip, engine status, and pilot command so investigators can piece together the story later.

### 🔹 File Name: IAuditRepository.cs
**Location**: `Ecom.Reporting.Application/Interfaces/IAuditRepository.cs`
**Purpose**: An interface that tells the system "I need to perform these DB operations," but doesn't care HOW they are done.

**FULL CODE**:
```csharp
using Ecom.Reporting.Domain.Entities;
namespace Ecom.Reporting.Application.Interfaces;

public interface IAuditRepository
{
    Task<IEnumerable<AuditLog>> GetByEntityAsync(string entityName, int entityId);
    Task<IEnumerable<AuditLog>> GetAllAsync(DateTime? from, DateTime? to);
```

**LINE-BY-LINE EXPLANATION**:

### 🔹 File Name: AuditService.cs
**Location**: `Ecom.Reporting.Application/Services/AuditService.cs`
**Purpose**: Orchestrates the audit workflows. Maps entities to DTOs.

**FULL CODE**:
```csharp
using AutoMapper;
using Ecom.Reporting.Application.DTOs;
using Ecom.Reporting.Application.Interfaces;
using Ecom.Reporting.Domain.Entities;

namespace Ecom.Reporting.Application.Services;

public class AuditService : IAuditService
{
    private readonly IAuditRepository _repo;
    private readonly IMapper _mapper;

    public AuditService(IAuditRepository repo, IMapper mapper)
    {
        _repo = repo;
        _mapper = mapper;
    }

    public async Task<IEnumerable<AuditLogDto>> GetByProductIdAsync(int productId)
    {
        var logs = await _repo.GetByEntityAsync("Product", productId);
        return _mapper.Map<IEnumerable<AuditLogDto>>(logs);
    }

    public async Task<IEnumerable<AuditLogDto>> GetAllAsync(DateTime? from, DateTime? to)
    {
        var logs = await _repo.GetAllAsync(from, to);
        return _mapper.Map<IEnumerable<AuditLogDto>>(logs);
    }

    public async Task WriteAsync(AuditLogDto dto)
    {
        var entity = _mapper.Map<AuditLog>(dto);
        await _repo.AddAsync(entity);
    }
}
```

**LINE-BY-LINE EXPLANATION**:
- `private readonly ...`: Store dependencies.
- `_mapper.Map`: Converts Domain Entity to DTO (and vice-versa). Automates the work of copying properties.
- `WriteAsync`: Takes a DTO from a Consumer, converts it to an Entity, and tells the Repository to save it.

## 🛠️ INFRASTRUCTURE LAYER (IMPLEMENTATION)

### 🔹 File Name: AuditRepository.cs
**Location**: `Ecom.Reporting.Infrastructure/Repositories/AuditRepository.cs`
**Purpose**: Raw SQL/EF logic to talk to the database.

**FULL CODE**:
```csharp
using Ecom.Reporting.Application.Interfaces;
using Ecom.Reporting.Domain.Entities;
using Ecom.Reporting.Infrastructure.Persistence;
using Microsoft.EntityFrameworkCore;

namespace Ecom.Reporting.Infrastructure.Repositories;

public class AuditRepository : IAuditRepository
{
    private readonly ReportingDbContext _context;

    public AuditRepository(ReportingDbContext context) => _context = context;

    public async Task<IEnumerable<AuditLog>> GetByEntityAsync(string entityName, int entityId) =>
        await _context.AuditLogs
            .Where(a => a.EntityName == entityName && a.EntityId == entityId)
            .OrderByDescending(a => a.CreatedAt)
            .ToListAsync();

    public async Task<IEnumerable<AuditLog>> GetAllAsync(DateTime? from, DateTime? to)
    {
        var query = _context.AuditLogs.AsQueryable();
        if (from.HasValue) query = query.Where(a => a.CreatedAt >= from.Value);
        if (to.HasValue) query = query.Where(a => a.CreatedAt <= to.Value);
        return await query.OrderByDescending(a => a.CreatedAt).ToListAsync();
    }

    public async Task AddAsync(AuditLog log)
    {
        await _context.AuditLogs.AddAsync(log);
        await _context.SaveChangesAsync();
    }

    public async Task<Dictionary<string, int>> GetActionCountsAsync()
    {
        return await _context.AuditLogs
            .GroupBy(a => a.Action)
            .Select(g => new { Action = g.Key, Count = g.Count() })
            .ToDictionaryAsync(x => x.Action ?? "Unknown", x => x.Count);
    }

    public async Task<IEnumerable<AuditLog>> GetFilteredAsync(DateTime? from, DateTime? to, string? reportType, int limit)
    {
        var query = _context.AuditLogs.AsQueryable();
        if (from.HasValue) query = query.Where(a => a.CreatedAt >= from.Value);
        if (to.HasValue) query = query.Where(a => a.CreatedAt <= to.Value);

        if (!string.IsNullOrWhiteSpace(reportType))
        {
            query = reportType.ToLower() switch
            {
                "catalog-quality" => query.Where(a => a.EntityName == "Product"),
                "low-stock" => query.Where(a => a.Action == "StockChanged"),
                "price-changes" => query.Where(a => a.Action == "PriceChanged"),
                "all-audit" => query,
                _ => throw new InvalidOperationException($"Unknown report type")
            };
        }

        return await query.OrderByDescending(a => a.CreatedAt).Take(limit).ToListAsync();
    }
}
```

**LINE-BY-LINE EXPLANATION**:
- `_context.AuditLogs`: Accesses the database table.
- `Where(...)`: Generates SQL `WHERE` clauses dynamically.
- `OrderByDescending`: Ensures newest records are at the top.
- `GroupBy`: Aggregates data on the DB side for performance.
- `SaveChangesAsync`: The final "Commit" that persists data to disk.

### 🔹 File Name: ProductApprovedConsumer.cs
**Location**: `Ecom.Reporting.Infrastructure/Messaging/Consumers/ProductApprovedConsumer.cs`
**Purpose**: Listens to RabbitMQ messages. When a product is approved in another microservice, this code wakes up.

**FULL CODE**: (Simplified structure for explanation)
```csharp
protected override async Task ProcessMessageAsync(string message)
{
    using var scope = _serviceProvider.CreateScope();
    var auditService = scope.ServiceProvider.GetRequiredService<IAuditService>();

    var eventData = JsonSerializer.Deserialize<ProductApprovedEvent>(message);
    
    await auditService.WriteAsync(new AuditLogDto
    {
        EntityName = "Product",
        EntityId = eventData.ProductId,
        Action = "Approved",
        EventType = "product.approved",
        SourceService = "WorkflowService"
    });
}
```

**LINE-BY-LINE EXPLANATION**:
- `CreateScope()`: Essential because this is a background thread. It creates a temporary "request context" to use the DbContext.
- `JsonSerializer.Deserialize`: Converts binary string from RabbitMQ into a C# object.
- `auditService.WriteAsync`: Sends the data to the DB via the application service.

# 🧠 STEP 3: LAYER UNDERSTANDING (CLEAN ARCHITECTURE)

1.  **Domain**: Contains `AuditLog` (What we record). No dependencies.
2.  **Application**: Contains `IAuditRepository` (How we want to record) and `AuditService` (The logic). Depends on Domain.
3.  **Infrastructure**: Contains `AuditRepository` (SQL implementation). Depends on Application/Domain.
4.  **API**: Contains `AuditController`. Coordinates everything.

# 🔄 STEP 4: COMPLETE REQUEST FLOW

1.  **Trigger**: Someone clicks "Approve" in the Workflow Service.
2.  **Messaging**: Workflow Service publishes a message to RabbitMQ topic `product.approved`.
3.  **Consumer**: `ProductApprovedConsumer` in Ecom.Reporting catches the message.
4.  **Application Logic**: Consumer calls `AuditService.WriteAsync`.
5.  **Infrastructure Logic**: `AuditService` calls `AuditRepository.AddAsync`.
6.  **Database**: Entity Framework runs `INSERT INTO AuditLogs (...)`.
7.  **Result**: The action is recorded forever.

# 🚀 STEP 9: SENIOR ENGINEER INSIGHTS

1.  **Optimization**: Use `AsNoTracking()` in repositories for Read-only queries to improve performance.
2.  **Scalability**: If the log volume grows to millions, consider **Database Partitioning** by `CreatedAt` (Year/Month).
3.  **Security**: Mask any sensitive data in `eventData` before logging to DB.
4.  **Logging**: Add Structured Logging (Serilog) inside Middleware for better observability.

# 🎯 STEP 10: INTERVIEW PREPARATION

**Q: Why use Clean Architecture for a simple reporting service?**
**A**: To protect business rules from technical churn. We can swap RabbitMQ for Kafka or MS SQL for PostgreSQL without rewriting a single line of business logic in the Domain or Application layers.

**Q: How do you handle database failures in consumers?**
**A**: We use RabbitMQ Acknowledgments. If `ProcessMessageAsync` fails (e.g., DB down), the message is NOT acknowledged and is requeued (Nack with requeue=true) to be retried automatically.

**Q: How is security handled?**
**A**: JWT Authentication. We validate the token signature in the API middleware and check for the "Admin" role on every request.
